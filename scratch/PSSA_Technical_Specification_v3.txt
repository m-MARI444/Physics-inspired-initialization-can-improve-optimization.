1. Foundational Neural Elements
1.1  Persistent Neuron Layer
Unlike memoryless static neurons, the PersistentNeuronLayer maintains a continuous internal state evolved via a gated recurrent write mechanism. The state encodes the neuron's semantic memory across arbitrary time horizons at O(1) memory cost.
1.1.1  State Concatenation
Given input x_t and previous state s_t, form the combined context vector:
Combined context:
c_t = [s_t; x_t]  ∈  ℝ^(2d)
1.1.2  Write Gate with Floor Clamping
The write gate determines how much new information overwrites the existing state. A floor clamp prevents permanent state lock by ensuring a minimum update rate:
Write gate (pre-clamp):
z_t = sigmoid(W_gate * c_t + b_gate)
Floor clamp (prevents state lock):
z_t = clamp(z_t, min = α_min)   where   α_min = 0.05
1.1.3  Candidate State and Recurrent Update
Candidate state:
\tilde{s}_t = tanh(W_cand * c_t + b_cand)
Recurrent update:
s_{t+1} = (1 − z_t) ⊙ s_t + z_t ⊙ \tilde{s}_t
Parameters: W_gate, W_cand ∈ ℝ^(d×2d) are linear projection weights. α_min = 0.05 guarantees minimum write rate preventing neuron saturation. The ⊙ operator denotes element-wise (Hadamard) product.
1.2  Surprise Normalization and Sigma Clamping
A critical stabilization mechanism introduced after Stage 30. As training progresses, the model becomes highly confident causing surprise standard deviation to decay toward zero, triggering spurious ignition cascades. The sigma clamp prevents this:
Sigma floor clamp:
σ_{S,new} = max(σ_{S,computed}, 0.05)
Adaptive ignition threshold:
τ_t = μ_S + 1.5 * σ_{S,new}
!  Biological analogy
The sigma clamp implements a minimum refractory period equivalent to biological neuron recovery time. Without it, any tiny fluctuation triggers ignition once the model becomes highly confident, causing the same cascade failure that occurred at Step 10590.

Routing vectors are organized by projecting node features onto a low-dimensional topological manifold M ⊂ ℝ^d_manifold (default d=3). This creates distance-based semantic affinities that are smooth, differentiable, and geometrically meaningful.

Project to manifold:

1.3.2  Pairwise Distance and Radial Basis Affinity
L2 pairwise distance:
D_ij = ||z_i − z_j||_2
Radial basis function affinity:
A_ij = exp(−γ * D_ij^2)
Row-normalized routing weight:
R_ij = A_ij / \sum_{k=1}^N A_ik
Where: R ∈ ℝ^(B×N×N) is the full routing weight matrix. γ ∈ ℝ is a learnable scaling parameter. The RBF kernel guarantees that nearby nodes in manifold space communicate more strongly, implementing a soft spatial prior over semantic similarity.

3.2  Hybrid Entity Allocation
Entity banks store persistent semantic objects across N_scopes x N_entities x N_timelines dimensions. Routing between focused (Top-1) and parallel (Top-2) allocation is controlled by semantic entropy:
Entropy-based routing weight:
\alpha_t = sigmoid(2.0 * (H_alloc − 1.2))

Interpretation: When H_alloc is high (ambiguous input), alpha_t → 1 and Top-2 parallel routing dominates, distributing the update across two candidate entities. When H_alloc is low (clear input), alpha_t → 0 and Top-1 focused routing dominates, committing to the single best entity match.

3.3  Multi-Tiered Episodic Memory & Prefetching
To scale PSSA's entity capacity (E) to 100k+ entities without GPU VRAM memory explosion or PCIe transfer bottlenecks, the architecture implements a multi-tiered memory hierarchy:
* L2 Active VRAM Bank: Stores the active entity slots currently loaded on the GPU.
* L3 Episodic Archive: A larger repository stored in CPU host memory.

3.3.1 LRU Swapping (Page-In / Page-Out)
For each token step t, the manager projects the query input x_t onto the manifold space:
q_t = W_q * x_t \in \mathbb{R}^{d_{manifold}}
We find the highest-scoring archived entity e_{arch} in L3 and the oldest active slot e_{evict} in L2. If the archived key's similarity exceeds the active similarity:
sim(q_t, e_{arch}) > sim(q_t, e_{evict})
we swap the two entities across the PCIe bus, paging e_{arch} into GPU VRAM and evicting e_{evict} to host RAM.

3.3.2 Look-Ahead Prefetching
Using the future predictions from Timeline 2 (E_{T2,t}), the manager projects future coordinate keys:
q_{future} = W_q * E_{T2,t} \in \mathbb{R}^{d_{manifold}}
It identifies if any future coordinates lie close to archived entities and asynchronously prefetches them into the L2 bank in the background, masking PCIe transfer latency.

9. Persistent State as Inference Bottleneck Solution
*  Core thesis
The strongest contribution of PSSA is not sparsity. It is the claim that persistent evolving state replaces token-history-based memory entirely. If this holds at scale, it directly attacks the largest inference bottleneck in modern LLMs: the need to carry and repeatedly process an ever-growing context window.
9.1  The Inference Bottleneck — Precisely Defined
The dominant cost of modern LLM inference is not the forward pass computation. It is memory bandwidth: the GPU must read the entire KV cache from VRAM on every generated token. At long contexts this dominates compute time by a factor of 3–10x on typical hardware.
The KV cache cost formula:
KV cache memory (grows linearly with context length L):
MemoryKV = 2 × nlayers × nheads × dhead × L × sizeof(dtype)
For a standard 7B model at float16, with 32 layers, 32 heads, and head dimension 128:
Concretely for 7B model:
MemoryKV = 2 × 32 × 32 × 128 × L × 2 bytes = 524,288 × L bytes
Empirical comparison point (Stage 30 benchmark):
At L = 7711:   524,288 × 7711 ≈ 4.04 GB of KV cache
PSSA state:   30,949 bytes   (constant, independent of L)
✓  Empirically measured
At 7711 BPE tokens: standard transformer KV cache = 44,600,424 KB (44.6 GB). PSSA persistent state = 30.949 KB. Ratio: 1,441,100× smaller. This ratio grows linearly with L — at 100k tokens PSSA's advantage is 18.7 million times smaller memory footprint.
9.2  Why Memory Bandwidth Is the Real Bottleneck
Modern GPUs have far more compute (FLOPS) than memory bandwidth (GB/s). The ratio is called arithmetic intensity. When arithmetic intensity of a workload falls below the hardware's compute-to-bandwidth ratio, the workload is memory-bound — adding more compute cores does nothing; the bottleneck is data movement.

Single-token autoregressive generation — which is what inference actually does — has arithmetic intensity near 1 for KV-cache attention. The GPU reads gigabytes of KV data to perform a small number of multiply-accumulates per token. PSSA's fixed-size state fits entirely in L2 cache on any modern GPU (~40MB). After the first token, every state read is a cache hit. Arithmetic intensity rises dramatically, the workload becomes compute-bound, and throughput scales with GPU count.
9.3  What the Persistent State Claim Requires
Replacing the KV cache with persistent state is a strong claim. It requires that the gated write mechanism can losslessly compress all causally relevant history into a fixed-dimensional vector. Three conditions must hold simultaneously:
9.3.1  Condition 1 — Sufficient representational capacity
The persistent state must have enough dimensions to encode all semantically distinct entities, relations, and temporal markers the task requires. The current PSSA state is d_model-dimensional per slot, with M slots per layer and L layers. Total state capacity:
State capacity:
Capacity = M × L × dmodel real-valued dimensions
For the Stage 30 configuration (M=5 slots, L=3 layers, d_model=64): 5 × 3 × 64 = 960 dimensions. This is sufficient for the TinyStories vocabulary and task complexity. The question at internet scale is whether this capacity can be increased without reintroducing the KV-cache-style linear growth. The answer is yes: increasing M or d_model scales capacity as O(M × L × d), which is a fixed constant independent of context length L.
9.3.2  Condition 2 — Stable long-horizon compression
The gated write mechanism must not allow semantic drift: the state encoding 'target = red box' at step 100 must still encode this correctly at step 10000. Stage 30 empirically demonstrated 89.76% cosine fidelity at 7711 steps. Three mechanisms enforce this:
Write gate floor (α_min = 0.05) prevents state from freezing, ensuring new evidence can always update the representation.
Surprise-driven reality anchoring resets the prospective timeline T2 to grounded T0 when contradiction score S_t > 0.40, preventing hallucination drift from compound errors.
Semantic inertia velocity tracking (v_k with β = 0.95 EMA) suppresses further updates to slots that have been changing rapidly, implementing a stability bias on converged representations.
Referent fidelity metric:
Fidelity(t) = cosine(sk,t, sk,0)

9.3.3  Condition 3 — Selective write preserves relevant history
The gated write must be selective: update the state when semantically important new information arrives, and preserve the state when information is redundant. The compound gate formula achieves this:
Compound selectivity gate:
zt = sigmoidWg(surprise) ⊙ softmaxWg(surprise)τ + δ̅s
High surprise → gate opens → new information overwrites. Low surprise (expected token) → gate stays at floor → state preserved. This is information-theoretically optimal compression for a stationary process: only encode what was not predicted. Transformers, by contrast, store every token regardless of whether it was predicted or not.

!  Why savings multiply, not add
Eliminating the KV cache (mechanism 2) removes the memory-bandwidth bottleneck that makes mechanism 1 (attention) slow even when theoretically O(k·n). Gated updates (mechanism 5) reduce the fraction of tokens that need full causal computation (mechanism 4). The compound effect at long contexts means PSSA's total inference cost stays constant while a transformer's grows without bound.
9.5  Where the Claim Must Still Be Proven
The persistent state claim is the strongest part of the proposal and the one that requires the most careful empirical validation. Three gaps remain between the current proof-of-concept and a full demonstration:
9.5.1  Semantic completeness at internet-scale vocabulary
TinyStories has a compact vocabulary with limited entity diversity. Internet-scale text introduces millions of distinct named entities, rare domain terms, ambiguous co-references, and long-range dependencies spanning thousands of tokens. The question is whether a fixed-capacity state can remain semantically complete across this diversity, or whether it begins to overwrite older but still relevant information.
Required experiment: train on Wikipedia (100M+ tokens, ∼50,000 distinct named entities). Measure referent fidelity at 1k, 10k, 50k, and 100k token horizons. If fidelity stays above 85% at 50k tokens, the persistent state claim holds for long-document tasks.
9.5.2  State capacity scaling law
For a fixed state size (M, L, d_model), there is a maximum task complexity the architecture can represent. As task complexity grows, the state will begin to overwrite useful older information with new information. The scaling law relating state capacity to maximum supportable context complexity is currently unknown.
Maximum supportable context length at given capacity:
Lmax(M, d) = ?   [unknown scaling law]
Required experiment: sweep M ∈ {5, 10, 20, 40} and d_model ∈ {64, 128, 256} on tasks of increasing complexity. Fit L_max(M,d) empirically. If L_max grows super-linearly in M×d, the architecture can be scaled to arbitrarily long contexts by increasing state capacity without reintroducing linear memory growth.
9.5.3  Compositional reasoning over compressed history
The hardest test is not memory retention but compositional reasoning: can the model answer questions that require combining facts from widely separated positions in the context? A transformer can do this because all past tokens are simultaneously accessible in the KV cache. PSSA must have encoded both facts into the persistent state in a form that supports their combination.
Required experiment: multi-hop question answering benchmarks (HotpotQA, MuSiQue) where the answer requires combining two facts separated by 1000+ tokens. Compare PSSA retrieval accuracy against a transformer with full context. If PSSA matches within 5%, the persistent state encoding is compositionally complete.
9.6  Theoretical Bound on Persistent State Advantage
Define the compression ratio CR as the ratio of PSSA state size to transformer KV cache size at context length L:
Compression ratio (grows with L):
CR(L) = M × Llayers × dmodel2 × nlayers × nheads × dhead × L = M × dmodel2 × nheads × dhead × L
The compression ratio grows linearly with L. At L = 1,441,100 (the empirically measured ratio from Stage 30), PSSA uses 1,441,100× less memory. This ratio is not a constant property of the architecture — it improves indefinitely as context grows. The longer the context, the greater the relative advantage of persistent state.
Limit behavior:
limL→∞CR(L) → ∞   [PSSA advantage unbounded as context grows]
✓  The core claim in one line
A transformer's inference cost is a function of context length. PSSA's inference cost is a function of state capacity, which is independent of context length. Every doubling of context length doubles the transformer's memory cost and roughly doubles its latency. PSSA's cost does not change. This is not a constant-factor improvement — it is an asymptotic advantage that grows without bound.
