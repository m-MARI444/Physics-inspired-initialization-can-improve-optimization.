2. Gap 1 — Language Projection Class Imbalance
2.1  Root cause
The language projection layer is a linear classifier mapping from the graph's output latent vector to action token logits. Its loss function is standard cross-entropy, which minimizes overall prediction error. When 90%+ of training frames have label Token 1 (arm stationary), the minimum-loss solution is to always predict Token 1, regardless of the latent state content. The layer learns a trivial constant function.
This is not a model capacity problem. The latent states correctly encode divergent world states — the classifier simply has no training signal to map those divergent states to non-dominant tokens.
2.2  Fix A — Focal loss (recommended primary fix)
Focal loss down-weights easy examples (the dominant Token 1 class) and focuses gradient signal on hard examples (rare action tokens). It modifies cross-entropy with a modulating factor:
FL(p_t) = -alpha_t * (1 - p_t)^gamma * log(p_t)
Parameters: gamma = 2.0 (standard starting point; increase to 5.0 if Token 1 dominance persists). alpha_t = inverse class frequency weight for class t. For a class that appears 90% of the time, alpha_t = 0.1. For a class appearing 1% of the time, alpha_t = 1.0.
import torch.nn.functional as F
def focal_loss(logits, targets, gamma=2.0, alpha=None):
    ce = F.cross_entropy(logits, targets, weight=alpha, reduction='none')
    pt = torch.exp(-ce)  # probability of correct class
    focal = (1 - pt) ** gamma * ce
    return focal.mean()
# Compute alpha from training set class frequencies
class_counts = torch.bincount(all_training_labels)
alpha = 1.0 / (class_counts.float() + 1e-6)
alpha = alpha / alpha.sum()  # normalize
2.3  Fix B — Separate language projection training phase
The language projection layer should be trained in two stages: (1) freeze the world model, train only the projection layer with focal loss on a class-balanced subset; (2) unfreeze and fine-tune end-to-end. This prevents the world model's gradients from being corrupted by the imbalanced language signal during early training.
2.4  Fix C — Oversampling rare action classes
Construct a balanced mini-batch sampler that ensures each token class appears with equal frequency within each batch, regardless of its frequency in the full dataset. This is complementary to focal loss and often more effective when class imbalance exceeds 50:1.
from torch.utils.data import WeightedRandomSampler
sample_weights = alpha[all_training_labels]  # per-sample weight
sampler = WeightedRandomSampler(sample_weights,
                                num_samples=len(dataset),
                                replacement=True)
!  Recommended implementation order
Apply Fix B first (separate training phase) — it costs nothing and immediately gives the projection layer clean gradient signal. Then add Fix A (focal loss) in the end-to-end fine-tune phase. Fix C (oversampling) is optional but useful if Token 1 still dominates after A+B.
3. Gap 2 — Proving Semantic Correctness of Divergence
3.1  The problem: distance is necessary but not sufficient
The Stage 5 benchmark measures ||s_A - s_B|| = 1.9375 and declares success. But this only proves the states are numerically different. It does not prove:
Universe A's state encodes the concept 'moved left'
Universe B's state encodes the concept 'moved right'
The divergence is caused by the action input rather than stochastic noise
Two random vectors in high-dimensional space are almost always far apart. A distance of 1.9375 could be produced by a model that ignores actions entirely and simply samples different noise each time.
3.2  Fix — Semantic correctness probing
Add a probing classifier that is trained (separately, on frozen world model states) to predict the action taken from the resulting latent state. If the world model is encoding actions correctly, the probe should achieve high accuracy. If it cannot, the divergence is spurious.
class ActionProbe(nn.Module):
    def __init__(self, state_dim, n_actions):
        super().__init__()
        self.probe = nn.Linear(state_dim, n_actions)  # single linear layer
    def forward(self, state):
        return self.probe(state)

# Low accuracy  => divergence is from noise, not actions
3.3  Fix — Counterfactual consistency test
Run the following extended benchmark:
Step 1. Prime model with state s_0.
Step 2. Apply action A_left for T=5 steps, record final state s_A.
Step 3. Reset to s_0. Apply action A_right for T=5 steps, record s_B.
Step 4. Reset to s_0. Apply A_left for T=5, then apply A_right for T=5. Record s_C.
Step 5. Reset to s_0. Apply A_right for T=5, then A_left for T=5. Record s_D.
Expected if causally correct: s_A != s_B (already proven), s_C should be close to s_D (left-then-right should approximate right-then-left in a symmetric environment). If s_C and s_D are far apart, the model is path-dependent in ways that do not reflect the physics.
!  Why this matters
A world model that produces spurious divergence is not useful for planning. If the model cannot reliably encode which action was taken in the latent state, it cannot simulate counterfactual futures for decision-making. This test is the gate between 'the latent states are different' and 'the model understands causality'.
4. Gap 3 — KL Posterior Collapse Prevention
4.1  The problem: fixed KL weight causes collapse
The current loss stack includes a KL divergence term L_KL that regularizes the Gaussian latent space. With a fixed weight beta, the optimizer finds the path of least resistance: it sets sigma(s,a) = 1 for all inputs, making z_t a sample from the prior N(0,1) rather than an informed posterior. When this happens, z_t carries zero information about the state and action — the stochastic component becomes pure noise.
This is called KL posterior collapse and it is the most common failure mode in VAE-style training. At low environment complexity (two actions, simple state), the model may not collapse because the dynamics function D can learn to ignore z_t entirely and still pass the divergence benchmark. But as the action space and state complexity grow, collapse becomes likely.
4.2  Fix A — Cyclical KL annealing (recommended)
Anneal the KL weight beta cyclically: start at 0, ramp to target, hold, then reset. Each cycle forces the model to first learn good reconstructions (beta=0, no KL pressure), then compress them (beta=target). The cycle prevents collapse by repeatedly releasing KL pressure before the posterior can fully converge to the prior.
beta(t) = beta_max * min(1, t mod T_cycle / T_ramp)
T_cycle = 1000   # steps per cycle
T_ramp  = 400    # ramp-up duration within each cycle
beta_max = 0.1   # start low; increase if latent space is too noisy
def beta_schedule(step):
    pos_in_cycle = step % T_cycle
    return beta_max * min(1.0, pos_in_cycle / T_ramp)
4.3  Fix B — Free bits (KL floor)
Set a minimum KL value per latent dimension below which no penalty is applied. This allows the model to use the first N bits of latent capacity 'for free' before KL pressure activates. It directly prevents the collapse mode where all dimensions converge to the prior.
L_KL_free = max(0, KL(q(z|s,a) || p(z)) - lambda_free)
Recommended value: lambda_free = 0.5 nats per dimension. With a 16-dimensional z_t, this allows 8 nats of free KL capacity — enough to encode meaningful action-conditional information.
4.4  Fix C — Monitoring sigma collapse
Add a diagnostic that logs the mean and variance of sigma(s,a) across the training batch at each epoch. If mean sigma approaches 1.0 across all dimensions, posterior collapse is occurring. This should be a standard training monitor, not an afterthought.
# In training loop
mu, log_sigma = encoder(state, action)
sigma = torch.exp(log_sigma)
metrics['sigma_mean'] = sigma.mean().item()
metrics['sigma_min']  = sigma.min().item()
metrics['kl_per_dim'] = kl_divergence(mu, sigma).mean(0).tolist()
# Alert if collapsing
if metrics['sigma_mean'] > 0.95:
    print('WARNING: KL posterior collapse detected')
5. Gap 4 — Action Distinguishability Hardening
5.1  The problem: zero-vector null action is brittle
The current L_action loss penalizes cases where D(s,a) is too similar to D(s,0), using the zero vector as a proxy for 'no action'. This works when zero is a reserved encoding, but fails in three cases:
Zero is a valid action in the action encoding scheme
The action embedding learns to map meaningful actions near zero
The environment has a 'null' action that genuinely produces no state change
5.2  Fix — Contrastive action loss over action pairs
Replace the zero-reference penalty with a pairwise contrastive loss over action pairs. For each training step, sample a second action a' != a from the batch. Penalize similarity between D(s,a) and D(s,a') in proportion to the semantic distance between a and a'.
L_action_v2 = max(0, sim(D(s,a), D(s,a')) - margin * dist(a, a'))
def contrastive_action_loss(states, actions, dynamics_fn, margin=0.5):
    # Sample negative action for each element in batch
    perm = torch.randperm(len(actions))
    neg_actions = actions[perm]
    
    pos_states = dynamics_fn(states, actions)      # D(s, a)
    neg_states = dynamics_fn(states, neg_actions)  # D(s, a')
    
    # Cosine similarity between imagined states
    sim = F.cosine_similarity(pos_states, neg_states)
    
    # Action distance (Euclidean in action embedding space)
    action_dist = F.pairwise_distance(actions, neg_actions)
    
    # Loss: push states apart in proportion to action difference
    loss = F.relu(sim - margin * action_dist.clamp(0, 1))
    return loss.mean()
i  Why this is more robust
The pairwise approach does not depend on any special encoding of 'null action'. It works for continuous, discrete, and mixed action spaces. It also naturally scales: actions that are very different (large dist(a,a')) must produce very different states, while actions that are similar may produce similar states.
